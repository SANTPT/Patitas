# Historia de Usuario
## ID
[FE-26]
## Título
Dashboard Usuario — Datos de compra seguros y gestión de pedidos
## Descripción
**Como** usuario registrado que realiza compras frecuentes en la tienda  
**Se desea** guardar mis datos de envío de forma segura en mi perfil y consultar el historial de mis pedidos desde el dashboard  
**Para** agilizar futuras compras y tener siempre a mano el estado de mis envíos.
## Criterios de Aceptación
### Escenario 1: Guardar datos de envío
```gherkin
Dado que el usuario navega a "/dashboard/usuario/perfil" sección "Datos de compra"
Cuando introduce su dirección de envío (calle, ciudad, código postal, país) y la guarda
Entonces la dirección queda guardada en su perfil
Y puede guardar hasta 3 direcciones con etiquetas (Casa, Trabajo, Otro)
Y puede marcar una dirección como "predeterminada"
```
### Escenario 2: Pre-rellenar el checkout
```gherkin
Dado que el usuario tiene una dirección predeterminada guardada
Cuando accede al proceso de checkout en "/checkout"
Entonces los campos de dirección de envío deben pre-rellenarse automáticamente con la dirección predeterminada
Y el usuario puede modificarlos antes de confirmar el pedido
```
### Escenario 3: Seguridad de datos de pago
```gherkin
Dado que el usuario navega a la sección de "Datos de compra"
Cuando ve la sección de métodos de pago
Entonces debe ver un badge visible de "Datos protegidos con cifrado SSL"
Y un aviso claro: "Los datos de tarjeta nunca se almacenan en nuestros servidores. Se procesan de forma segura a través de nuestra pasarela de pago."
Y el único dato guardado es el nombre del titular (nunca número de tarjeta, CVV ni fecha de caducidad)
```
### Escenario 4: Historial de pedidos en el dashboard
```gherkin
Dado que el usuario navega a "/dashboard/usuario/pedidos"
Cuando la vista carga
Entonces debe ver todos sus pedidos ordenados por fecha descendente con estado coloreado, total y productos
Y debe haber un botón "Ver detalle" que lleva a "/pedido/:id"
Y un botón "Cambio / Devolución" que lleva a "/devoluciones?order=PT-XXXXX"
```
### Escenario 5: Eliminar datos de compra
```gherkin
Dado que el usuario quiere eliminar sus datos de envío guardados
Cuando hace clic en "Eliminar todos mis datos de compra"
Entonces debe aparecer un diálogo de confirmación con advertencia
Y al confirmar, todos sus datos de envío guardados se eliminan
Y la sección queda vacía con opción de volver a añadir
```
## Notas
- Los datos de dirección se guardan en `user.shippingAddresses` (array). Solo accesibles para el propio usuario.
- Los datos de tarjeta NUNCA se almacenan: la pasarela (Stripe/Redsys) gestiona todo mediante tokenización.
- El checkout debe leer `authStore.user.shippingAddresses[defaultIndex]` para pre-rellenar (FE-26 depende de FE-21).
- La sección "Datos de compra" es invisible para Admin y Super Admin (solo visible en rol "user").
- En backend: `PATCH /api/usuarios/:id/addresses`, `DELETE /api/usuarios/:id/addresses`.
