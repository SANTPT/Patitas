
# Historia de Usuario
## ID
[FE-05]
## Título
Página de Login — autenticación de usuarios
## Descripción
**Como** padre registrado en Patitas  
**Se desea** poder iniciar sesión con correo y contraseña  
**Para** acceder a su perfil, historial de recursos guardados y funciones de la comunidad.
## Criterios de Aceptación
### Escenario 1: Renderizado del formulario de login
```gherkin
Dado que el usuario navega a la ruta "/login"
Cuando la página carga
Entonces debe ver un formulario centrado con: campo email, campo contraseña (tipo password), checkbox "Recordarme", botón "Iniciar sesión" y enlace "¿Olvidaste tu contraseña?"
Y el formulario debe estar sobre un fondo con la imagen fondo_morado.png o fondo_azul.png del proyecto
```
### Escenario 2: Validación de campos vacíos
```gherkin
Dado que el usuario hace clic en "Iniciar sesión" sin completar los campos
Cuando el formulario intenta enviarse
Entonces deben mostrarse mensajes de error debajo de cada campo vacío: "El email es obligatorio" y "La contraseña es obligatoria"
Y el borde del campo inválido debe cambiar a rojo (#e74c3c)
Y el foco debe posicionarse en el primer campo con error
```
### Escenario 3: Formato de email inválido
```gherkin
Dado que el usuario escribe un email con formato incorrecto (sin "@" o sin dominio)
Cuando pierde el foco del campo (evento blur)
Entonces debe mostrarse el mensaje "Ingresa un email válido" debajo del campo
Y el campo debe mostrar borde rojo hasta que se corrija el valor
```
### Escenario 4: Login exitoso
```gherkin
Dado que el usuario ingresa credenciales válidas
Cuando hace clic en "Iniciar sesión"
Entonces el botón debe mostrar un estado de carga (spinner o texto "Cargando...")
Y tras la respuesta exitosa de la API, debe redirigirse al dashboard o página de inicio con sesión activa
Y el nombre del usuario debe aparecer en el header en lugar del botón "Iniciar sesión"
```
### Escenario 5: Credenciales incorrectas
```gherkin
Dado que el usuario ingresa credenciales incorrectas
Cuando el servidor responde con error 401
Entonces debe mostrarse un mensaje de error general: "Email o contraseña incorrectos"
Y los campos deben limpiarse solo en la contraseña, manteniendo el email
Y no debe revelarse cuál de los dos campos es incorrecto (seguridad)
```
## Notas
- Usar `reactive()` o `ref()` de Vue 3 para el estado del formulario.
- El estado de carga debe deshabilitar el botón para evitar doble envío.
- En fases futuras incluir login con Google (OAuth 2.0).
- El token de sesión (JWT) debe guardarse en `httpOnly cookie` o `localStorage` según la política de seguridad del equipo.
- Agregar enlace a la página de registro: "¿No tienes cuenta? Regístrate".
