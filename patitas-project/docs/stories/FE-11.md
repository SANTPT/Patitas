
# Historia de Usuario
## ID
[FE-11]
## Título
Perfil de usuario — gestión de datos y recursos guardados
## Descripción
**Como** usuario autenticado de Patitas  
**Se desea** poder ver y editar su perfil, y acceder a los recursos guardados  
**Para** personalizar su experiencia y retomar fácilmente el contenido que le resultó útil.
## Criterios de Aceptación
### Escenario 1: Vista del perfil
```gherkin
Dado que el usuario autenticado navega a "/perfil"
Cuando la página carga
Entonces debe ver su avatar (o iniciales si no tiene foto), nombre completo, email, y fecha de registro
Y debe haber pestañas para: "Mis datos", "Recursos guardados" y "Mis pedidos"
```
### Escenario 2: Edición de datos personales
```gherkin
Dado que el usuario está en la pestaña "Mis datos"
Cuando hace clic en "Editar perfil"
Entonces los campos nombre y avatar deben volverse editables
Y al guardar, debe enviarse PATCH /api/usuarios/:id y mostrarse un toast de confirmación "Perfil actualizado"
```
### Escenario 3: Recursos guardados
```gherkin
Dado que el usuario ha guardado recursos desde la sección de recursos
Cuando navega a la pestaña "Recursos guardados"
Entonces debe ver la lista de recursos marcados con opción de eliminar cada uno
Y si no tiene guardados, debe mostrarse el mensaje "Aún no has guardado recursos. ¡Explora la sección de recursos!"
```
### Escenario 4: Cambio de contraseña
```gherkin
Dado que el usuario quiere cambiar su contraseña
Cuando completa el formulario de "Cambiar contraseña" (contraseña actual + nueva + confirmar)
Entonces el sistema debe validar que la contraseña actual sea correcta antes de permitir el cambio
Y debe mostrarse confirmación y cerrar la sesión en otros dispositivos
```
## Notas
- El avatar puede ser una URL de imagen o generarse con las iniciales del nombre en un círculo de color lila.
- Los recursos guardados deben persistir en el backend, no solo en localStorage.
- La pestaña "Mis pedidos" es dependiente del módulo de tienda (FE-08).
