
# Historia de Usuario
## ID
[FE-14]
## Título
Blog — listado y detalle de artículos informativos
## Descripción
**Como** padre buscando orientación  
**Se desea** leer artículos escritos por profesionales sobre autismo, crianza y desarrollo  
**Para** tomar decisiones más informadas sobre el acompañamiento de su hijo.
## Criterios de Aceptación
### Escenario 1: Listado del blog
```gherkin
Dado que el usuario navega a "/blog"
Cuando la página carga
Entonces debe ver las publicaciones en formato de cards con imagen de portada, título, autor, fecha, categoría y tiempo de lectura estimado
Y las publicaciones deben estar ordenadas por fecha descendente
```
### Escenario 2: Detalle de artículo
```gherkin
Dado que el usuario hace clic en un artículo
Cuando navega a "/blog/:slug"
Entonces debe ver el contenido completo renderizado desde Markdown o HTML
Y debe haber una sección de "Artículos relacionados" al final con 3 sugerencias
Y un botón de "Guardar artículo" para usuarios autenticados
```
### Escenario 3: Compartir en redes sociales
```gherkin
Dado que el usuario está leyendo un artículo
Cuando hace clic en los botones de compartir (Facebook, WhatsApp, X)
Entonces debe abrirse el diálogo nativo de compartir de cada red con la URL y título del artículo pre-completados
```
### Escenario 4: Tiempo de lectura estimado
```gherkin
Dado que un artículo está cargado
Cuando el componente se monta
Entonces debe calcularse el tiempo de lectura basado en el conteo de palabras del contenido (promedio: 200 palabras por minuto)
Y debe mostrarse como "Lectura: X min" en la cabecera del artículo
```
## Notas
- El contenido del blog puede venir del backend o de un CMS headless (Contentful, Sanity) en fases futuras.
- Implementar meta tags dinámicos (Open Graph) para SEO con `useHead()` o `vue-meta`.
- Considerar comentarios en artículos como extensión futura de la funcionalidad de comunidad.
