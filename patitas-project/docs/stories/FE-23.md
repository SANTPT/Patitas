# Historia de Usuario
## ID
[FE-23]
## Título
Dashboard Admin — Gestión de perfiles de niños y registro de sesiones
## Descripción
**Como** terapeuta o centro de atención temprana  
**Se desea** poder crear perfiles para los niños que atiendo, registrar cada sesión terapéutica y actualizar su progreso general  
**Para** llevar un historial estructurado y compartirlo con las familias de forma privada.
## Criterios de Aceptación
### Escenario 1: Crear perfil de niño
```gherkin
Dado que el Admin navega a "/dashboard/admin/ninos/nuevo"
Cuando completa el formulario con nombre, fecha de nacimiento, diagnóstico y foto (opcional)
Entonces el perfil se crea y aparece en su listado de niños
Y se genera un código de invitación para que el padre lo vincule desde su cuenta
Y el perfil es privado por defecto: solo visible para el centro y los padres vinculados
```
### Escenario 2: Registrar sesión de terapia
```gherkin
Dado que el Admin está en el perfil de un niño
Cuando hace clic en "Nueva sesión"
Entonces debe ver un formulario con: fecha, duración, tipo de terapia, objetivos trabajados (con escala 1-5 por objetivo) y observaciones libres
Y al guardar, la sesión aparece en el historial del niño ordenada por fecha
Y el padre vinculado puede ver el historial desde su dashboard
```
### Escenario 3: Actualizar avance general
```gherkin
Dado que el Admin está en el perfil de un niño
Cuando hace clic en "Actualizar avance" en el panel de radar
Entonces debe poder ajustar los valores de 4-6 dimensiones de desarrollo (comunicación, social, cognitivo, motor, autonomía)
Y al guardar, el gráfico de radar del niño se actualiza
Y se guarda la fecha de la actualización para mostrar el histórico de evolución
```
### Escenario 4: Vincular padre al perfil
```gherkin
Dado que el Admin creó un perfil de niño
Cuando el padre introduce el código de invitación desde su cuenta de usuario
Entonces el perfil del niño queda vinculado a la cuenta del padre
Y el padre puede ver el progreso del niño desde su dashboard de usuario
```
## Notas
- El gráfico de radar usa Chart.js (disponible en el proyecto) o similar.
- Los datos de niños se almacenan en el store `children.js` (nuevo) con mock en localStorage bajo `patitas_mock_children`.
- Las sesiones se guardan en `patitas_mock_sessions` con referencia al id del niño.
- En backend: `POST /api/children`, `GET /api/children` (filtrado por centro), `POST /api/children/:id/sessions`, `PATCH /api/children/:id/progress`.
