# Historia de Usuario
## ID
[FE-15]
## Título
Página de Gestión de Pedidos — Cambios, Devoluciones y Envíos
## Descripción
**Como** cliente que realizó una compra  
**Se desea** poder consultar el estado de mi pedido y gestionar cambios, devoluciones o incidencias sin necesidad de bajar al footer ni escribir un email  
**Para** resolver cualquier problema de forma autónoma y guiada en menos de 2 minutos.
## Criterios de Aceptación
### Escenario 1: Localización del pedido
```gherkin
Dado que el usuario navega a "/devoluciones"
Cuando escribe su código de pedido (ej: PT-48291) en el campo de búsqueda
Entonces el sistema debe mostrar una tarjeta con los datos del pedido (productos, fecha, estado y total)
Y debe habilitarse el botón "Continuar con este pedido"
Y si el código no existe, debe mostrarse un mensaje de error descriptivo con link a contacto
```
### Escenario 2: Selección de motivo
```gherkin
Dado que el usuario localizó su pedido correctamente
Cuando avanza al paso 2
Entonces debe ver 6 tarjetas de motivo: dañado/defectuoso, producto incorrecto, cambio de opinión, cambio de variante, no llegó, otro
Y cada tarjeta debe mostrar el tipo de resolución que aplica (devolución gratuita, cambio urgente, etc.)
Y al seleccionar una, debe avanzar automáticamente al formulario correspondiente
```
### Escenario 3: Formulario contextual
```gherkin
Dado que el usuario seleccionó un motivo
Cuando el formulario carga
Entonces debe mostrar únicamente los campos relevantes para ese motivo (ej: campo de fotos solo para "dañado")
Y el usuario debe poder seleccionar qué producto(s) de su pedido están afectados
Y el botón de envío debe estar deshabilitado hasta que se completen los campos obligatorios
```
### Escenario 4: Confirmación y pasos siguientes
```gherkin
Dado que el usuario completó el formulario y envió la solicitud
Cuando la solicitud se registra exitosamente
Entonces debe ver una pantalla de confirmación con los pasos exactos que se van a seguir según el tipo de solicitud
Y debe recibir confirmación del email al que se notificará el estado
```
### Escenario 5: Acceso desde la página de pedido
```gherkin
Dado que el usuario está en "/pedido/:id" viendo su pedido
Cuando hace clic en "Solicitar devolución o cambio"
Entonces debe navegar a "/devoluciones?order=PT-XXXXX&tab=devoluciones" con el pedido y la pestaña pre-rellenados
```
## Notas
- El flujo es de 3 pasos: localizar pedido → elegir motivo → formulario contextual → confirmación.
- Los 6 motivos mapean a tipos de resolución: `devolucion`, `cambio`, `incidencia`, `consulta`.
- La política de plazos (14 días devolución, 30 días cambio) debe mostrarse de forma clara junto al formulario.
- En backend: conectar con `POST /api/requests` con campos `{ orderId, type, subReason, productIds, description, photos, contactEmail }`.
- Las fotos se enviarán como FormData multipart cuando exista el backend.
