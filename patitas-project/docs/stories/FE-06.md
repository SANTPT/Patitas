
# Historia de Usuario
## ID
[FE-06]
## Título
Página de Registro — creación de cuenta de padre/tutor
## Descripción
**Como** nuevo visitante de Patitas  
**Se desea** poder crear una cuenta con sus datos  
**Para** acceder a recursos personalizados, guardar contenido y unirse a la comunidad.
## Criterios de Aceptación
### Escenario 1: Formulario de registro completo
```gherkin
Dado que el usuario navega a "/registro"
Cuando la página carga
Entonces debe ver campos para: nombre completo, email, contraseña, confirmar contraseña, y checkbox de aceptación de términos
Y debe haber un botón "Crear cuenta" y un enlace "¿Ya tienes cuenta? Inicia sesión"
```
### Escenario 2: Validación de contraseña segura
```gherkin
Dado que el usuario está escribiendo su contraseña
Cuando el campo tiene foco
Entonces debe mostrarse un indicador visual de fortaleza (débil / media / fuerte) basado en longitud y caracteres especiales
Y la contraseña debe tener mínimo 8 caracteres
```
### Escenario 3: Coincidencia de contraseñas
```gherkin
Dado que el usuario ha completado ambos campos de contraseña
Cuando pierde el foco del campo "Confirmar contraseña"
Entonces si los valores no coinciden debe mostrarse "Las contraseñas no coinciden" en rojo
Y el botón "Crear cuenta" debe permanecer deshabilitado hasta que todas las validaciones pasen
```
### Escenario 4: Registro exitoso
```gherkin
Dado que el usuario completa el formulario correctamente y acepta los términos
Cuando hace clic en "Crear cuenta"
Entonces debe recibir un correo de verificación y ver un mensaje de confirmación en pantalla
Y debe redirigirse a una página de "Revisa tu correo" o directamente al onboarding
```
## Notas
- El checkbox de términos debe enlazar a la página `/terminos` en una nueva pestaña.
- Considerar añadir campo opcional de "¿Cómo nos conociste?" para métricas de adquisición.
- La contraseña nunca debe enviarse en texto plano; asegurar HTTPS en producción.
