# Historia de Usuario
## ID
[FE-21]
## Título
Dashboard — Roles y estructura base (Super Admin, Admin, Usuario)
## Descripción
**Como** equipo de desarrollo  
**Se desea** tener una estructura de dashboard con rutas protegidas por rol, layout compartido y navegación lateral contextual  
**Para** construir encima las funcionalidades específicas de cada tipo de usuario de forma organizada.
## Criterios de Aceptación
### Escenario 1: Acceso al dashboard según rol
```gherkin
Dado que un usuario autenticado navega a "/dashboard"
Cuando el guard del router evalúa su rol
Entonces debe redirigir a "/dashboard/super-admin" si su rol es "superadmin"
Y a "/dashboard/admin" si su rol es "admin"
Y a "/dashboard/usuario" si su rol es "user"
Y si no está autenticado, debe redirigir a "/login?redirect=/dashboard"
```
### Escenario 2: Layout del dashboard
```gherkin
Dado que el usuario está en cualquier ruta del dashboard
Cuando la vista carga
Entonces debe ver un sidebar izquierdo con los ítems de navegación relevantes para su rol
Y un topbar con su nombre, avatar y botón de cerrar sesión
Y el área de contenido principal a la derecha
Y en móvil, el sidebar debe colapsarse en un menú hamburguesa
```
### Escenario 3: Guard de rol
```gherkin
Dado que un usuario con rol "user" intenta acceder directamente a "/dashboard/super-admin"
Cuando el guard evalúa la ruta
Entonces debe redirigir a la vista de dashboard que le corresponde ("/dashboard/usuario")
Y debe mostrarse un toast informativo "No tienes permisos para acceder a esa sección"
```
### Escenario 4: Edición de perfil propio (todos los roles)
```gherkin
Dado que cualquier usuario autenticado navega a "/dashboard/perfil"
Cuando accede a su perfil
Entonces debe poder editar nombre, foto de perfil, email y contraseña
Y los cambios deben reflejarse en el header de la app al guardar
```
## Notas
- Estructura de rutas propuesta: `/dashboard` → redirect por rol; `/dashboard/super-admin`, `/dashboard/admin`, `/dashboard/usuario`.
- El componente `DashboardLayout.vue` envuelve a todos con sidebar + topbar.
- El sidebar lee `authStore.user.role` para renderizar los ítems de navegación.
- Meta en el router: `meta: { requiresAuth: true, roles: ['superadmin'] }` para proteger secciones.
- La edición de perfil reutiliza `authStore.updateProfile()` y `authStore.changePassword()` ya implementados.
