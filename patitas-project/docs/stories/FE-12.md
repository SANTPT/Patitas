
# Historia de Usuario
## ID
[FE-12]
## Título
Configuración de Vue Router y navegación protegida
## Descripción
**Como** equipo de desarrollo  
**Se desea** tener el sistema de rutas configurado con guards de autenticación  
**Para** que cada página sea accesible según el estado de sesión del usuario y la aplicación sea una SPA funcional.
## Criterios de Aceptación
### Escenario 1: Rutas públicas y privadas definidas
```gherkin
Dado que el proyecto usa Vue Router 4
Cuando se define el archivo router/index.js
Entonces deben existir rutas públicas (/, /recursos, /tienda, /centros, /login, /registro) y rutas privadas (/perfil, /comunidad/nuevo, /carrito/pago)
Y cada ruta privada debe tener meta: { requiresAuth: true }
```
### Escenario 2: Guard de autenticación global
```gherkin
Dado que un usuario no autenticado intenta acceder a una ruta privada
Cuando el guard beforeEach evalúa la ruta
Entonces debe redirigirse a "/login" con query param redirect="/ruta-original"
Y tras el login exitoso debe redirigirse automáticamente a la ruta original
```
### Escenario 3: Rutas con parámetros dinámicos
```gherkin
Dado que el usuario navega a "/tienda/42"
Cuando Vue Router procesa la ruta
Entonces el componente ProductoDetalle debe recibir el parámetro id como prop o useRoute().params.id
Y si el producto no existe (404 de la API), debe redirigirse a una página de error 404 personalizada
```
### Escenario 4: Página 404 personalizada
```gherkin
Dado que el usuario escribe una URL que no existe en la aplicación
Cuando Vue Router no encuentra coincidencia
Entonces debe renderizarse la vista NotFound.vue con el logo de Patitas, mensaje amigable y botón "Volver al inicio"
```
## Notas
- Usar `createRouter` con `createWebHistory` (HTML5 mode) para URLs limpias sin `#`.
- El estado de autenticación debe consultarse desde el store de Pinia, no de localStorage directamente.
- Agregar `scrollBehavior` para que las rutas nuevas comiencen desde el top de la página.
- Lazy loading en rutas pesadas (tienda, comunidad, centros) con `() => import(...)`.
