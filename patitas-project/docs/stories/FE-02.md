
# Historia de Usuario
## ID
[FE-02]
## Título
Hero banner con contenido dinámico y badge de video
## Descripción
**Como** padre que visita el sitio por primera vez  
**Se desea** ver una sección de bienvenida visualmente atractiva con información clave del portal  
**Para** entender de inmediato qué es Patitas y sentirse en un lugar de confianza.
## Criterios de Aceptación
### Escenario 1: Layout de dos columnas en desktop
```gherkin
Dado que el usuario accede desde desktop
Cuando carga la página de inicio
Entonces la sección hero debe mostrar texto a la izquierda e imagen a la derecha en grid de dos columnas iguales
Y el título debe usar fuente Fredoka con la palabra destacada en color lila (#c58cf2) con subrayado decorativo
```
### Escenario 2: Video badge de los patos
```gherkin
Dado que el componente HeroBanner está montado
Cuando el DOM está listo
Entonces el video "vido_patitas.mp4" debe reproducirse automáticamente, sin sonido, en loop y en modo playsinline
Y debe estar contenido en un círculo con borde blanco de 5px, ubicado en la esquina inferior derecha de la imagen
Y debe tener animación de float suave (translateY) continua
```
### Escenario 3: Estadísticas de confianza
```gherkin
Dado que el usuario visualiza la sección hero
Cuando lee el área inferior del lado de texto
Entonces debe ver 3 estadísticas: "+2.4k Familias", "120+ Recursos" y "35+ Centros"
Y cada estadística debe mostrar el número en grande con Fredoka y la etiqueta en gris pequeño debajo
```
### Escenario 4: Adaptación a móvil
```gherkin
Dado que el usuario accede desde un móvil
Cuando la pantalla es menor a 860px
Entonces el layout debe cambiar a una sola columna con texto centrado
Y el chip flotante debe ocultarse para no obstruir la imagen
Y el video badge debe reubicarse a la esquina inferior derecha dentro del card de imagen
```
## Notas
- El fondo usa `fondo_azul.png` con `background-size: cover`.
- El chip flotante "Diagnóstico temprano" tiene animación independiente de float.
- Los CTAs "Comenzar ahora" y "Conocer recursos" deben tener href configurables via props o router-link en versiones futuras.
- El wave de transición inferior debe coincidir exactamente con el color `#f0f8ff` de la sección siguiente.
