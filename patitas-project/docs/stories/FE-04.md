
# Historia de Usuario
## ID
[FE-04]
## Título
Footer global con newsletter, links y métodos de pago
## Descripción
**Como** usuario del sitio  
**Se desea** tener un footer completo con links útiles, redes sociales y opción de suscripción  
**Para** encontrar información legal, de contacto y mantenerse al día con las novedades de Patitas.
## Criterios de Aceptación
### Escenario 1: Layout de tres columnas en desktop
```gherkin
Dado que el usuario llega al final de cualquier página
Cuando visualiza el footer
Entonces debe ver 3 columnas: (1) logo + nombre + tagline + redes sociales, (2) links de Recursos y Ayuda, (3) newsletter + métodos de pago
Y el fondo debe usar fondo_morado.png con overlay de gradiente azul-lila
```
### Escenario 2: Formulario de newsletter
```gherkin
Dado que el usuario escribe su email en el campo del newsletter
Cuando hace foco en el input
Entonces el contenedor debe resaltarse con borde lila y sombra sutil (box-shadow rgba 197,140,242)
Cuando hace clic en el botón de envío
Entonces el sistema debe prevenir el comportamiento por defecto (@submit.prevent) y disparar la lógica de suscripción
```
### Escenario 3: Redes sociales con hover
```gherkin
Dado que el usuario pasa el cursor sobre un ícono de red social
Cuando se activa el estado hover
Entonces el círculo debe cambiar su fondo a lila (#c58cf2) y el ícono a blanco
Y debe haber una elevación de 3px y una sombra lila (rgba 197,140,242, 0.45)
```
### Escenario 4: Barra de copyright inferior
```gherkin
Dado que el footer está renderizado
Cuando el usuario llega al final
Entonces debe ver una barra separada por border-top con el copyright "© 2026 Patitas" a la izquierda y links legales (Aviso Legal, Privacidad, Términos, Cookies) a la derecha
Cuando la pantalla es menor a 560px
Entonces la barra debe colapsar a layout vertical centrado
```
## Notas
- El wave superior del footer debe usar el color `#f0f8ff` para hacer la transición limpia desde la sección de cards.
- Los links generados con `v-for` desde `linkCols` facilitan agregar nuevas secciones.
- Los íconos SVG de redes sociales están inline para evitar dependencias externas.
- En fases futuras, el newsletter debe conectarse a un endpoint `/api/newsletter/subscribe`.
