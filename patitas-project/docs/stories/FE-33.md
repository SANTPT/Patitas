# Historia de Usuario
## ID
[FE-33]
## Título
Chatbot flotante — Asistente de IA con Gemini API (gratuita)
## Descripción
**Como** visitante o usuario de la plataforma  
**Se desea** tener un botón flotante de chat en todas las páginas que abra un asistente de IA especializado en autismo y atención temprana  
**Para** resolver dudas rápidas sobre la plataforma, recursos o el proceso de diagnóstico sin necesidad de contactar al equipo.
## Criterios de Aceptación
### Escenario 1: Botón flotante visible en toda la plataforma
```gherkin
Dado que el usuario está en cualquier página de Patitas
Cuando la página carga
Entonces ve un botón circular flotante en la esquina inferior derecha con ícono de chat
Y el botón tiene un tooltip "¿Necesitas ayuda?"
Y el botón NO aparece dentro del propio chat abierto
```
### Escenario 2: Abrir y cerrar el chat
```gherkin
Dado que el usuario hace clic en el botón flotante
Cuando el chat se abre
Entonces aparece una ventana de chat sobre el botón con: cabecera con nombre "Patitas Asistente", área de mensajes y campo de texto con botón de enviar
Y hay un mensaje de bienvenida automático del bot
Y el botón flotante cambia a ícono de cerrar (✕)
Cuando hace clic en ✕
Entonces el chat se cierra con animación y el botón vuelve al estado inicial
```
### Escenario 3: Enviar un mensaje y recibir respuesta de Gemini
```gherkin
Dado que el usuario escribe un mensaje y presiona Enter o el botón de enviar
Cuando el mensaje se procesa
Entonces el mensaje del usuario aparece alineado a la derecha con su avatar
Y aparece un indicador de "escribiendo..." mientras Gemini procesa
Y la respuesta del bot aparece alineada a la izquierda con el logo de Patitas
Y el bot responde en español con un tono amable y empático
```
### Escenario 4: Prompt del sistema (contexto de Patitas)
```gherkin
Dado que el chatbot se inicializa
Cuando se hace la primera llamada a Gemini
Entonces el prompt del sistema incluye: rol del asistente (experto en autismo y atención temprana Patitas), tono (amable, empático, claro), límites (no dar diagnósticos médicos, derivar a profesionales para casos clínicos)
Y el historial de la conversación se envía en cada llamada para mantener contexto
```
### Escenario 5: Límites y manejo de errores
```gherkin
Dado que la API de Gemini devuelve un error o no responde
Cuando el timeout o error ocurre
Entonces el bot muestra: "Lo siento, ahora mismo no puedo responder. Por favor inténtalo de nuevo o contáctanos en /contacto"
Y el campo de texto se rehabilita para que el usuario pueda volver a escribir
Dado que el usuario lleva 20 mensajes en la sesión
Cuando envía el mensaje 21
Entonces el bot avisa: "Hemos llegado al límite de esta conversación. ¿Quieres empezar una nueva?" con botón de reinicio
```
### Escenario 6: Persistencia de la sesión del chat
```gherkin
Dado que el usuario navegó a otra página con el chat abierto
Cuando la nueva página carga
Entonces el historial de la conversación se mantiene (guardado en memoria del componente global)
Y al cerrar y volver a abrir el chat, el historial se conserva durante la misma sesión del navegador
```
## Notas
- El componente `ChatbotWidget.vue` se registra globalmente en `App.vue` para aparecer en todas las páginas.
- Se usa **Gemini 1.5 Flash** (tier gratuito de Google AI Studio) vía `fetch` directo al endpoint de la API.
- La API key de Gemini se guarda en `.env` como `VITE_GEMINI_API_KEY` (nunca en el código).
- El historial se mantiene en un `ref([])` dentro del componente; se limita a los últimos 20 turnos para no superar el contexto.
- El prompt del sistema establece que el bot es "Pati", asistente de Patitas, especializado en autismo, atención temprana y uso de la plataforma.
- **No almacenar conversaciones en localStorage** por privacidad; se pierden al recargar la página.
- El botón flotante tiene `z-index: 999` para estar siempre visible.
- En fases futuras: integrar historial en el backend para usuarios autenticados, analytics de preguntas frecuentes.
