
# Historia de Usuario
## ID
[FE-07]
## Título
Sección de Recursos — listado con filtros por categoría
## Descripción
**Como** padre buscando información específica  
**Se desea** explorar los recursos del portal filtrando por tipo (artículos, guías, videos, centros)  
**Para** encontrar rápidamente lo que se necesita sin revisar todo el contenido.
## Criterios de Aceptación
### Escenario 1: Listado de recursos paginado
```gherkin
Dado que el usuario navega a "/recursos"
Cuando la página carga
Entonces debe ver una grilla de cards de recursos con imagen, título, categoría, descripción corta y botón "Ver más"
Y deben mostrarse máximo 9 recursos por página con paginación inferior
```
### Escenario 2: Filtros por categoría
```gherkin
Dado que el usuario está en la página de recursos
Cuando hace clic en un filtro de categoría (Artículos / Guías / Videos / Centros)
Entonces la grilla debe actualizarse mostrando solo los recursos de esa categoría
Y el filtro activo debe resaltarse con fondo lila (#c58cf2)
Y la URL debe actualizarse con el query param ?categoria=articulos para permitir compartir el filtro
```
### Escenario 3: Buscador de recursos
```gherkin
Dado que el usuario escribe en el buscador de la sección
Cuando ha escrito al menos 3 caracteres
Entonces los resultados deben filtrarse en tiempo real sin recargar la página
Y si no hay resultados debe mostrarse el mensaje "No encontramos recursos para tu búsqueda" con sugerencias
```
### Escenario 4: Estado de carga
```gherkin
Dado que el sistema está obteniendo recursos de la API
Cuando la petición está en curso
Entonces deben mostrarse skeleton loaders en lugar de las cards
Y el buscador y filtros deben estar deshabilitados durante la carga
```
## Notas
- Usar `computed()` de Vue 3 para el filtrado reactivo local.
- La paginación puede ser client-side (si el total es menor a 100) o server-side con query params.
- Cada card de recurso debe tener un tag de color que coincida con los colores de acento del proyecto.
