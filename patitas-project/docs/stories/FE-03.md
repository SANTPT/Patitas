
# Historia de Usuario
## ID
[FE-03]
## Título
Sección de cards informativas con CTA de comunidad
## Descripción
**Como** padre recién llegado al sitio  
**Se desea** ver tarjetas con los temas principales del portal (autismo, atención temprana, comunidad)  
**Para** acceder rápidamente a la información que se necesita según la situación.
## Criterios de Aceptación
### Escenario 1: Renderizado de las tres cards
```gherkin
Dado que el componente InfoCards está montado
Cuando se renderiza la sección
Entonces deben aparecer 3 cards en grid de 3 columnas con los títulos: "QUÉ ES EL AUTISMO", "GUÍA DE ATENCIÓN TEMPRANA" y "COMUNIDAD PATITAS"
Y cada card debe tener emoji identificador, tag de categoría, descripción y botón "Leer más"
```
### Escenario 2: Color de acento por categoría
```gherkin
Dado que cada card tiene un color de acento asignado mediante la variable CSS "--card-accent"
Cuando se renderiza la card
Entonces el fondo del área superior debe ser un tono muy suave del color de acento (12% de opacidad sobre blanco)
Y el tag de categoría, el borde del emoji y el botón deben usar el color de acento correspondiente
```
### Escenario 3: Hover en cards
```gherkin
Dado que el usuario pasa el cursor sobre una card
Cuando se activa el estado hover
Entonces la card debe elevarse 6px con una sombra más pronunciada
Y la transición debe usar cubic-bezier con efecto elástico suave (.34,1.56,.64,1)
```
### Escenario 4: Banner CTA de diagnóstico
```gherkin
Dado que el usuario llega al final de la sección de cards
Cuando visualiza el banner inferior
Entonces debe ver el texto "¿Acabas de recibir un diagnóstico?" con subtítulo y botón "Únete a la comunidad"
Y el banner debe usar fondo_morado.png con overlay de gradiente para legibilidad
Y el botón debe tener fondo blanco con texto azul oscuro (#1a5b82)
```
## Notas
- Las cards se generan con `v-for` desde un array reactivo, facilitando agregar más cards en el futuro.
- En la versión con router, el botón "Leer más" debe ser `<router-link>` apuntando a la ruta correspondiente.
- En mobile (< 860px), el grid debe colapsar a 1 columna con ancho máximo de 460px centrado.
- El banner CTA en mobile debe cambiar a layout vertical (flex-direction: column).
