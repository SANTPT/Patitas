# Historia de Usuario
## ID
[FE-19]
## Título
Comunidad — Tabs de Grupos, Eventos e Historias funcionales
## Descripción
**Como** miembro de la comunidad Patitas  
**Se desea** que las tabs de Grupos, Eventos e Historias de Éxito de la sección comunidad tengan contenido real o simulado y acciones funcionales  
**Para** explorar la comunidad más allá del foro de posts y no encontrar pantallas vacías.
## Criterios de Aceptación
### Escenario 1: Tab Grupos — visualización
```gherkin
Dado que el usuario hace clic en la tab "Grupos"
Cuando el contenido carga
Entonces debe ver al menos 2 grupos predefinidos con nombre, descripción, número de miembros y botón "Explorar grupo"
Y cada grupo debe mostrar si es público o privado
```
### Escenario 2: Tab Grupos — acción "Explorar grupo"
```gherkin
Dado que el usuario no autenticado hace clic en "Explorar grupo"
Cuando el botón se activa
Entonces debe redirigir a "/login?redirect=/comunidad" con mensaje "Inicia sesión para acceder a los grupos"
Y si el usuario está autenticado, debe mostrar un toast informativo de que los grupos completos estarán disponibles próximamente
```
### Escenario 3: Tab Eventos
```gherkin
Dado que el usuario hace clic en la tab "Eventos"
Cuando el contenido carga
Entonces debe ver al menos 2 eventos próximos con título, fecha, modalidad (presencial/online) y botón "Apuntarse"
Y el botón "Subscribirse a boletín de eventos" debe funcionar: si autenticado → toast de confirmación; si no → redirigir a registro
```
### Escenario 4: Tab Historias de Éxito
```gherkin
Dado que el usuario hace clic en "Historias"
Cuando el contenido carga
Entonces debe ver al menos 2 historias de familias con foto de perfil (avatar SVG), nombre y resumen de la historia
Y los botones "Crear cuenta" y "Saber más" deben tener destinos reales: "/registro" y "/recursos"
```
### Escenario 5: Link "Hablar con un guía"
```gherkin
Dado que el usuario ve el botón "Hablar con un guía" en la comunidad
Cuando hace clic en él
Entonces si no está autenticado, debe redirigir a "/login?redirect=/comunidad"
Y si está autenticado, debe navegar a "/contacto?asunto=guia"
```
## Notas
- El `handleAction()` actual solo redirige a login o dispara un toast genérico — debe refinarse por acción.
- Los datos de grupos, eventos e historias se simulan con datos mock en el componente (no requieren API propia por ahora).
- En backend: `GET /api/comunidad/grupos`, `GET /api/comunidad/eventos`, `GET /api/comunidad/historias`.
- El botón "Ver todos los grupos" debe navegar a `/comunidad?tab=grupos` o abrir un panel expandido.
